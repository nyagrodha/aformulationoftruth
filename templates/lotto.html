1. Register (client-side, Tor-friendly)

Client computes commitment and stores secret locally before submit.

secret = random()
nullifier = Poseidon(secret, "nullify")
C = Poseidon(secret, nullifier)
POST /lotto/commit { commitment: C }

2. Close (T1, operator-signed)

Server builds deterministic Merkle tree from sorted commitments and signs the fairness anchor.

root R = Merkle(sorted(C_i))
anchor = H(R, N, drand_round_target)
signature = Sign_operator(anchor)

3. Draw (T2, public)

Anyone can recompute winner from published root and drand beacon output.

B = drand(round_target)
winner_idx = H(R, B) mod N

4. Claim (T3, {{ claim_window_days }}-day window)

Winner submits secret/nullifier material plus a valid Merkle proof for the winning commitment.

POST /lotto/claim
{
  "round_id": 7,
  "commitment": "C_winner",
  "nullifier": "nullifier",
  "leaf_index": 3,
  "merkle_proof": ["hash1", "hash2", "..."]
}

5. Audit (always possible)

Audit endpoint returns round metadata, sorted commitments, and spent nullifiers.
