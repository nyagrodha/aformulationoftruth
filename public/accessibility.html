Accessibility

Our commitment to making self-inquiry accessible to everyone, regardless of ability or circumstance.

Our Commitment

We believe that the practice of self-reflection should be available to all. This site strives to meet WCAG 2.1 Level AA guidelines, with many Level AAA enhancements where possible.

Perceivable

Content is presented in ways that all users can perceive, with text alternatives, captions, and sufficient color contrast.

Operable

All functionality is available via keyboard. No time limits on responses. Clear navigation and focus indicators.

Understandable

Content is readable and predictable. Forms provide clear error messages and instructions.

Robust

Compatible with current and future assistive technologies. Semantic HTML throughout.

Accessibility Features

Navigation

  • Skip links: Bypass navigation and jump directly to main content
  • Keyboard navigation: All interactive elements are reachable and operable via keyboard
  • Focus indicators: Visible focus rings on all interactive elements
  • Logical heading structure: Proper heading hierarchy (h1 through h6)

Visual Design

  • Color contrast: Minimum 4.5:1 ratio for normal text, 3:1 for large text (WCAG AA)
  • Color independence: Information is not conveyed by color alone
  • Resizable text: Text scales up to 200% without loss of content
  • Dark mode: Respects prefers-color-scheme system preference
  • Reduced motion: Animations disabled when prefers-reduced-motion is set

Content

  • Plain language: Questions are written clearly without unnecessary jargon
  • Text alternatives: Meaningful alt text for images; decorative images marked appropriately
  • No time pressure: Take as long as you need to answer questions
  • Error prevention: Ability to review and correct responses before submission

Technical

  • Semantic HTML: Proper use of landmarks, headings, lists, and form elements
  • ARIA labels: Additional context provided where native HTML is insufficient
  • Form accessibility: Labels associated with inputs; error messages linked to fields
  • Language declaration: Page language specified for screen readers

Assistive Technology Compatibility

This site has been tested with:

  • VoiceOver (macOS/iOS)
  • NVDA (Windows)
  • Keyboard-only navigation
  • High contrast mode
  • Screen magnification (up to 400%)

Voice Input: For users who prefer voice input, we recommend Handy, a free, offline speech-to-text tool that respects your privacy.

Known Limitations

We are actively working to address the following areas:

  • Some decorative elements may not be fully optimized for screen readers
  • Video content on the home page lacks audio descriptions
  • Some third-party content (embedded fonts) may have limited accessibility

Feedback & Contact
